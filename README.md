# ICCU Case Competition – Fair Lending Risk Dashboard

## Overview
This project originated as part of the Idaho Central Credit Union (ICCU) Case Competition, where my team analyzed lending data to identify potential **Fair Lending Law disparities** and compliance risks.

The initial analysis was conducted primarily using **Python (Jupyter Notebook)**, focusing on exploratory data analysis and static visualizations to uncover patterns in loan outcomes.

After the competition, I revisited the project to expand its impact by developing a **fully interactive Power BI dashboard**, transforming the original analysis into a more accessible, dynamic, and decision-focused tool.

---

## Objective
The goal of this project was to:
- Identify disparities in lending outcomes across applicant groups  
- Detect potential compliance risks related to Fair Lending regulations  
- Translate exploratory analysis into an **interactive dashboard for stakeholders**  
- Communicate insights clearly through intuitive and user-driven visualizations  

---

## Project Evolution

### Phase 1: Exploratory Analysis (Python)
- Conducted data cleaning and preprocessing  
- Performed exploratory analysis to identify trends and anomalies  
- Built static visualizations to evaluate approval rates and potential disparities  

### Phase 2: Dashboard Development (Power BI)
- Re-designed the analysis into an interactive reporting experience  
- Developed dynamic visuals and KPI tracking for easier interpretation  
- Implemented filters and drill-down capabilities for deeper exploration  
- Focused on improving **clarity, usability, and storytelling**

---

## Dataset
- Lending dataset containing applicant and loan information  
- Features include (anonymized):
  - Loan outcome (approved/denied)  
  - Applicant financial attributes (e.g., income, loan amount)  
  - Demographic indicators (aggregated and anonymized)  

 Note: All data used in this project is anonymized and intended for analysis purposes only.

---

## Tools & Technologies
- **Power BI** – dashboard development and visualization  
- **Python (Jupyter Notebook)** – exploratory data analysis  
- **SQL** – data querying and preparation  

---

## Dashboard Features

### 1. Anomaly Detection & Loan Profiling
- Dedicated dashboard tab for identifying anomalous lending decisions  
- Defined anomalies as:
  - Loans that were **denied despite strong financial indicators** (e.g., low DTI and high FICO score)  
  - Loans that were **approved despite higher risk profiles** (e.g., high DTI and low FICO score)  
- Displays detailed attributes for each flagged loan to support deeper investigation  
- Enables filtering by **city** to explore geographic patterns in anomalies  

### 3. Prohibited Factor Analysis
- Secondary dashboard tab focused on **breakdown of anomalies by prohibited factors**  
- Highlights which factors contribute most to anomalous lending decisions  
- Enables prioritization of high-impact risk areas for compliance review  
- Supports investigation into potential sources of bias in decision-making  

### 6. Interactive Exploration
- Filters and drill-downs for:
  - Applicant attributes  
  - Geographic segmentation (city-level filtering)

## Key Insights
- Identified differences in approval rates across applicant groups that may warrant further investigation  
- Translated complex statistical findings into a format accessible to non-technical stakeholders  
- Demonstrated how interactive dashboards can support regulatory risk monitoring
